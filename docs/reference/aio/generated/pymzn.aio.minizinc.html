pymzn.aio.minizinc¶

pymzn.aio.minizinc(mzn, *dzn_files, args=None, data=None, include=None, stdlib_dir=None, globals_dir=None, declare_enums=True, allow_multiple_assignments=False, keep=False, output_vars=None, output_base=None, output_mode='dict', solver=None, timeout=None, two_pass=None, pre_passes=None, output_objective=False, non_unique=False, all_solutions=False, num_solutions=None, free_search=False, parallel=None, seed=None, rebase_arrays=True, keep_solutions=True, return_enums=False, max_queue_size=0, **kwargs)¶

Coroutine version of the pymzn.minizinc function.

Parameters:max_queue_size (int) – Maximum number of solutions in the queue between the solution parser and the returned solution stream. When the queue is full, the solver execution will halt untill an item of the queue is consumed. This option is useful for memory management in cases where the solution stream gets very large and the caller cannot consume solutions as fast as they are produced. Use with care, if the full solution stream is not consumed before the execution of the Python program ends it may result in the solver becoming a zombie process. Default is 0, meaning an infinite queue.
